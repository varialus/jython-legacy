  • Tries Python modules (source in .py files). Searches along sys.path dir list. Python packages are subdirectories of sys.path dirs with a __init__.py file whose execution on package import populates the corresponding first-class module. 
    >>> import xml
>>> xml
<module xml at 22396194>
>>> xml.__file__
'/home/pedronis/jython-2.1/Lib/xml/__init__.py'
>>> import xml.sax
    ( .py files are implicitly and automatically compiled to java bytecode and cached as $py.class files)
  • Tries Java importing. Search along classpath + sys.path with Java rules. Jar files are scanned, as necessary, on startup to cache information about contained java packages (and classes). Beyond that one can explicitly register packages: 
    >>> import sys ; sys.add_package("foo")
<java package foo at 11829808>
>>> import foo
    Classloading happens normally from jython code classloader and possibly a secondary classloader (sys.classLoader).
